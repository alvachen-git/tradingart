```text
任务/功能：Godot 客户端首版联调
严重级别（S1/S2）：S2
判定依据：客户端窗口可启动但脚本编译失败，页面空白；后端与原系统数据未受影响，属于局部功能不可用且有替代路径（可继续用 Streamlit 版）。
处理时限要求：当前任务结束前完成记录并修复可用性。

错误现象：Godot 启动后画面空白，控制台报 `Could not find type "ApiClient"`、`RunState not declared` 等解析错误。
影响范围：仅影响 `godot_client` 前端展示与交互；不影响原 `K线训练` 和既有 K 线数据读写。

根因分析：`main.gd` 直接使用自定义类型注解（`ApiClient/RunState/KlineChartView`），在当前 Godot 运行方式下全局 class 注册未及时可见；同时项目将脚本 warning 视为 error，部分 `:=` 推断为 Variant 触发编译失败。
修复动作：改为 `preload(...).new()` + 非自定义类型注解；对关键变量补显式类型（`Variant/float/int`）消除 warning-as-error。
防复发动作（必须可执行）：新增 Godot 修改后固定执行一次 `Godot --headless --log-file /tmp/godot-headless.log --path <project> --quit` 作为脚本编译门禁。
验证方式：无头启动无 Script Parse/Compile Error；GUI 启动后出现登录与操作控件而非空白窗口。
```

### 记录 2（S2）

```text
日期：2026-02-19
任务/功能：Godot 客户端创建新局联调
严重级别（S1/S2）：S2
判定依据：客户端可启动但“新开一局”出现请求失败，属于局部功能不可用；后端服务与原 K 线主流程未受影响。
处理时限要求：当前任务内修复并验证。

错误现象：点击“新开一局”后状态显示 `http request failed: 44`，终端报 `HTTPRequest is processing a request`。
影响范围：Godot 客户端的对局创建与状态刷新链路；不影响既有 Streamlit 版。

根因分析：`api_client.gd` 复用单个 `HTTPRequest` 节点处理所有请求，当存在重叠请求时触发 Godot `ERR_BUSY(44)`。
修复动作：改为每次请求创建独立 `HTTPRequest` 实例，完成后释放；并在 API 客户端增加请求串行槽位与 `request_result` 失败兜底，避免重叠点击与空响应造成误判。
防复发动作（必须可执行）：客户端 API 层禁止共享同一个活跃 `HTTPRequest`，并在回归时执行“快速连点”场景验证。
验证方式：连点“新开一局/刷新局状态”不再出现 `ERR_BUSY(44)`，接口均返回正常业务结果。
```
